6---什么是索引?SQL Server 2000里有什么类型的索引?----------------

简单地说，索引是一个数据结构，用来快速访问数据库表格或者视图里的数据。在SQL Server里，它们有两种形式:聚集索引和非聚集索引。
聚集索引在索引的叶级保存数据。这意味着不论聚集索引里有表格的哪个(或哪些)字段，这些字段都会按顺序被保存在表格。由于存在这种排序，所以每个表格只会有一个聚集索引。
非聚集索引在索引的叶级有一个行标识符。这个行标识符是一个指向磁盘上数据的指针。它允许每个表格有多个非聚集索引。

13---你觉得在数据库的约束的修改中，需要注意一些什么问题？
如果是在表上添加新列，可以像刚建表时一样，直接指定该列的约束。  ALTER TABLE Student ADD Score DECIMAL(3,1) DEFAULT(18)
但是除了not null 约束以外，给表上已有列添加约束，就不能直接使用alter column，比如这样写就不行， alter table students alter column age int default(18);
必须使用 add constraint;
比如 alter table students
add constraint pk_students_id primary key(id);////最后（）里面写要加约束的列名
primary,unique,check写法都一样，都是最后在（）里面写上要加约束的列名，
而default约束写法不同，
ALTER TABLE Student
ADD CONSTRAINT DF_Age DEFAULT 18 FOR Age; -- 注意写法的差异是在最后写for**;**代表列名

16--请你聊一聊三大范式，它是什么，有什么作用？
范式就是规范样式，就是数据库表设计时应该遵守的规范，应该采取的样式。
范式的目的就是减少冗余，保证数据的一致性。
第一范式：要求我们拆分属性，保证表中的每一列值不可再分解.
第二范式：消除非主属性对于码的部分函数依赖，就是要求：一行数据非主键列都只对主键完全依赖。
第三范式：在第二范式的基础上，消除了非主属性对于主键的传递函数依赖。

20-事务的四大特性是什么？
1，原子性，事务作为一个整体不可再分，事务中的所有内容要完成都全部完成，要不完成都不完成，不能只完成一部分。
2，一致性，事务的运行不能破坏数据的一致性，比如各种约束。
3，隔离性，正在执行的事务不能被其他事务干扰。
4，持久性，一旦完成，改变就是永久性的，比如commit之后就不能rollback了。
5，隐式事务，需要显示的被打开，set implicit_transactions on. 还记得完成操作后要记得关闭，set implicit_transactions off;

1，委托，什么是委托？
答：委托是一种特殊的类型，代表的是方法的引用，所以需要指定参数和返回值，委托类型的变量可以像方法一样使用，
另外委托类型的变量可以像普通变量一样，可以被赋值，作为方法参数传递，作为方法返回值。
2，.net为我们预置了哪两种委托，他们有什么区别？
答：预置了Func和Action两种委托，区别是func有返回值的方法，使用泛型参数依次表示方法的参数，最后
一个表示方法的返回值，action是没有返回值的方法，使用泛型参数依次表示方法参数。
3，事件，什么是事件？
事件从基本上说就是一个用户操作，比如按键，点击，鼠标移动等等动作。或者是一些提示信息，如系统生成的通知。
应用程序需要在事件发生时响应事件。例如，中断。
4，请你说一下匿名方法的的语法特点？
答：匿名方法的语法特点有，找不到方法名，由关键字delegate代替，匿名方法不需要标志返回值类型。
5，什么是闭包？
答：闭包是一种语法现象，使用匿名方法就产生了外部方法不能使用匿名函数中的变量，
匿名方法能够使用并改变其外部方法的变量或者参数。

6,进程和线程的区别？
答：进程是系统进行资源分配和调度的单位；线程是CPU调度和分派的单位，一个进程可以有多个线程，这些线程共享这个进程的资源。
